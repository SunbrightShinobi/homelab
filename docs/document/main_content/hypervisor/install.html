1.5.1. VMware vSphere ESXi Deployment for gerudo¶

1.5.1.1. Establish an Intel AMT Remote Desktop session to gerudo¶

Prerequistes:

  • Download and installation of Meshcommander from MeshCommander Download

  • Intel AMT is configured per Dell Precision 3431, Intel Management Engine Configuration for gerudo

  1. From workstation, open MeshCommander

  2. Computer Management (Networked Computers Icon)

  3. Find gerudo-mgmt and to on the same row at right-hand side of window Connect

  4. Wait for system status to populate then Remote Desktop on left-hand side menu

  5. In Remote Desktop section of MeshCommander Connect

  6. You now have a remote desktop session to gerudo in its current state.

    Note

    During reboots or power cycles MeshCommander sometimes disconnects and has a delay to reconnect. Just reconnect as able but this is why its recommend to start from powered down for OS installations over Remote Desktop.

    Mark Complete:

    Hostname

    Print and Sign Name

    gerudo

1.5.1.2. Installation of VMware vSphere ESXi for gerudo¶

Important

Currently for VMware vSphere ESXi 7.0.0-15843807, PXE boot via BIOS or UEFI is non-operational

Important

Currently for VMware vSphere ESXi 7.0.0-15843807, customization of ISO file is non-functional as well as using Ventoy for a ISO booting USB.

Important

These steps are written from the expectation previous section is complete for establishing a remote desktop session to host via MeshCommander

  1. If host is powered on, Power Actions… –> Power down –> OK and wait for red notification that Remote computer is not powered on.

  2. IDER and configure the following options:

    • Source ISO & IMG from file

    • .ISO file Choose File and browse to location of and then select VMware-VMvisor-Installer-7.0.0-15843807.x86_64.iso

    • Start Immediate

  3. OK and verify you have have a green bar with IDE-R Session, Connected at top of MeshCommander now

  4. Power Actions… –> Power On to IDE-R CDROM –> OK

  5. Verify you see Dell boot image and then it continues to Loading ESXi installer

  6. At the Welcome to the VMware ESXI Installation screen, press Enter

  7. At the EULA screen, F11

  8. At the Select a Disk to Install or Upgrade, Local: Samsung Flash Drive (mpx.vmhba34:C0:T0:L0) and Enter

  9. If prompted, use UP and DOWN and SPACEBAR to select Install then Enter to perform a fresh install over current ESXi installation

  10. At the Please select a keyboard layout screen, select US Default and Enter

  11. <adminPassword> then reenter to confirm <adminPassword>

  12. At the Confirm Install screen, F11

  13. Once installation is complete, Stop IDE-R Session from MeshCommander

  14. Enter to Reboot

    Mark Complete:

    Hostname

    Print and Sign Name

    gerudo

1.5.1.3. Post-Installation Configuration of VMware vSphere ESXi for gerudo¶

  1. Verify system is booted up and ready at the Yellow & Black screen then F2 to Customize System

  2. Enter root for Login Name and <adminPassword> for Password then Enter

  3. Configure Management Network then Enter

    1. Network Adaptors then Enter

      • Verify that vmnic0 is [X] and Status is Connected

      • Verify that vmnic1-2 are [ ] and Status is Connected

      • Enter

    2. VLAN (optional) then Enter to apply settings and exit menu.

      Note

      It is recommended to have your VMware management network as the native vlan or EPG on the configured trunk or as an access port. Which is configured as blank for the VLAN ID.

      • Verify that VLAN ID (1-4094, or 4095 to access all VLANs): blank

    3. IPv4 Configuration and then Enter

      • Set static IPv4 address and network configuration and then Space to mark selection

      • IPv4 Address = 10.1.10.101

      • Subnet Mask = 255.255.255.0

      • Default Gateway = 10.1.10.1

      • Enter

    4. DNS Configuration and then Enter

      • Use the following DNS server addresses and hostname and then Space to mark selection

      • Primary DNS Server = 10.1.10.1

      • Alternate DNS Server = 0.0.0.0

      • Hostname = gerudo

      • Enter

    5. Custom DNS Suffixes and then Enter

      • Suffixes = hyrule.binarylandscapes.com

      • Enter

    6. ESC and Y to Apply changes and restart management network

  4. Test Management Network then Enter

    • Verify that 10.1.10.1 is listed for Ping Address #0

    • Enter to perform a ping test to the default Gateway

    • Verify you get Pinging address #0 (10.1.10.1).             OK.

    • Verify you get Pinging address #1 (10.1.10.1).             OK.

    • If populated Verify you get Pinging address #2 (0.0.0.0).             OK.

    • Verify you get Resolving hostname (gerudo.hyrule.binarylandscapes.com).             OK.

    • Enter

  5. Configure Management Network then Enter

    1. IPv6 Configuration and then Enter

      • Disable IPv6 (restart required) and then Space

      • Enter

    2. ESC and Y to Apply changes and reboot host

    Mark Complete:

    Hostname

    Print and Sign Name

    Date
